# Athlevo MCP Server

HTTP-only MCP server that exposes tools to the Athlevo AI Agent. All operations go through this server; the agent does not access databases or services directly.

## Setup

```bash
cd mcp-server
npm install
cp .env.example .env   # then set DATABASE_URL and optional PORT
npm run build
```

**Environment:** Configuration is read from `.env` (see `.env.example`). The repo ignores `.env`; copy from `.env.example` and set at least `DATABASE_URL` (Neon connection string). Optional: `PORT` (default 3002).

**Database (Neon + Drizzle):** To introspect the Neon DB and generate the Drizzle schema locally, run once (after setting `DATABASE_URL` in `.env`):

```bash
npm run db:pull
```

This runs `drizzle-kit pull` and writes schema into `src/db/schema/`. The server uses the Drizzle client from `src/db/index.ts` with that schema.

## Run

- **Production:** `npm start` (runs `dist/server.js`)
- **Development:** `npm run dev` (tsx watch with reload)

Server listens on **port 3002** by default. Override with `PORT`:

```bash
PORT=4000 npm start
```

## Transport

**HTTP only.** No stdio. The agent connects via HTTP (e.g. `http://localhost:3002`).

- **MCP endpoint:** `POST /mcp` — MCP JSON-RPC over Streamable HTTP
- **Health:** `GET /health` — returns `{ "ok": true }`

## Tools

### `events.search`

Search sports events.

**Input (optional):**

- `sport` (string) — filter by sport
- `location` (string) — filter by location

**Response (TOOLS_SPEC):**

- Success: `{ "success": true, "data": [...] }`
- Error: `{ "success": false, "error": "message" }`

**Example HTTP (conceptual)**

After the MCP client has initialized a session, a tool call is sent as part of the MCP protocol (e.g. `method: "tools/call"`, `params: { name: "events.search", arguments: { sport: "running" } }`). The server responds with the standardized payload in the tool result content.
